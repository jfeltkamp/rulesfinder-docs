# User Guide

## The Goal of a RuFi Project

RuFi is a participation and argumentation software. The goal of a RuFi project is therefore the creation of a complete set of rules or a contract, i.e. a collection of practical rules that apply to a group of people and should be followed in order to achieve _shared_ goals. The regulations should be recognized and supported by all affected persons as much as possible. Therefore, they must not appear arbitrary, but must be understandable to everyone, must obviously serve the achievement of the common goals by being derived from the necessity of problems that need to be solved for this purpose.

## Editing Workflow: Problem - Rule - Argumentation

_Problem_, _Rule_, and _Argumentation_ form a meaningful unit. A rule always refers to a problem. It should prevent the occurrence or the consequences of this problem through a practical instruction. 

Arguments serve to analyze precisely this relationship between _Problem and Rule_. They show, at each aspect of the relationship, to what extent a rule is _suitable_ or _unsuitable_ for solving the problem.
